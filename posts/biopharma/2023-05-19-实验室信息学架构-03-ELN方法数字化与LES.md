# 实验室信息学架构（03）ELN、方法数字化与 LES

> 把“实验记录”变为“结构化方法与数据”，是可复现与自动化的前提。本篇聚焦 ELN（电子实验记录）与 LES（方法执行系统）。

## 1. 从自由记录到结构化方法
- 自由文本→模板化→参数化→程序化（带校验与计算）
- 模板要素：目的、适用范围、仪器/耗材、试剂/标准品、步骤与参数、计算与判定、偏差与异常处理
- 版本化：方法/模板必须版本化，实验按版本实例化

## 2. 参数与步骤建模
- 参数类型：数值、枚举、布尔、文件、序列（多次采集）
- 约束：范围、精度、依赖、必填/条件必填
- 步骤：准备→校准→采集→计算→复核
- 计算：内置函数或脚本（受控），输出可追溯

## 3. LES：让“执行”可控、可证据化
- 步骤驱动：指令、采集、校验、判定、分支
- 即时校验：单位/范围/格式/逻辑校验
- 设备联动：触发仪器方法、读取通道数据、采集日志
- 证据沉淀：每步产出结构化记录与附件清单、自动生成审计与报告

## 4. 与 LIMS/SDMS 的配合
- 入口：LIMS 下达任务/样本清单
- 执行：LES 绑定方法版本与参数实例
- 产出：结果与文件（带元数据）推送回 LIMS / SDMS（建立可追踪链接）

## 5. 质量与合规（方法层）
- 双人复核/关键步骤双签
- 关键参数锁定与变更审批
- 计算脚本受控与验证证据
- 偏差捕捉与 CAPA 闭环

## 6. 最小模板示例（片段）
```json
{
  "template": "HPLC-Assay",
  "version": 3,
  "steps": [
    {"name": "校准", "params": [{"name": "标准溶液浓度", "type": "number", "unit": "mg/mL", "range": [0.8, 1.2]}]},
    {"name": "进样", "params": [{"name": "进样体积", "type": "number", "unit": "uL", "range": [5, 20]}]},
    {"name": "计算", "calc": "ASSAY = (峰面积/标准曲线)*100"}
  ]
}
```

## 7. 落地建议
- 从 1-2 个高频方法起步，优先硬约束（范围/单位），逐步引入脚本与仪器联动
- 用“参数-证据-结果”三元组检查可复现性
